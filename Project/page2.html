Activity 3

How would you design an interface for a 1000 floor elevator?


The design of the interface is like what is shown above, it only has 14 buttons, the 10 buttons are the numbers wherein the users/ people will push the button for their desired floor. The other 4 button are the “Emergency” if the user will push that button, the building admin will be notified right away that there is an emergency in a certain floor where the pushing button happens, the second button is the “Help” it is to give brief information on how the buttons are functioning. The third button is the “Cancel” button, if the user typed a wrong number, he/she can delete it right away. The last button is “Click to Cancel a Queued Floor” if the user changes his/her mind to go to the floor where he/she supposed to go, he/she can push the button then type the floor that he/she wants to cancel BUT it will notify everyone in the elevator that a certain floor will be cancelled so that if there is anyone will go to the floor that he/she will cancel will be alarmed so that it will be cancelled. If that happens, the user who wants to cancel a certain floor can cancel it by just pushing the “Cancel” button (the second function of the “Cancel” Button). This buttons or interface will be placed in different parts of the elevator, so that everyone can put their desired floor. If the user will enter the same time, it won’t affect the system because of the queueing system of data structures.
This is the big screen wherein the queued floors will be shown, actually it is big so that it can be seen by everyone inside the elevator.

Activity 4

Need Finding Activity

Objectives:
• To provide service to the condo unit owners
• To help segregate wastes coming from condo units
Problems:
• Majority of Condo owners nowadays do not have a time to clean their condo units
• Unsegregated Wastes coming from different condo units.
Propose Project:
• An Online Booking application that lets the user to book for a certain people to clean their condo units.
• This application is a bridge of two entities; the Cleaning Service Provider agencies and Condo Unit Owners or simply the Customers.
• The people that will be hired by the cleaning service provider agencies will be trained to know how to properly segregate the waste coming from the condo units.
• This application will generate profit in terms of advertisements.
• This application will give customers the ability to choose from which company to prioritize using rate system.
• This application can negate false booking.
Related Applications:

• Grab • Angkas

Those applications are related to our proposed project because it also books people to do service to the customer but in terms of transportation.

• GoodWork.ph

This is a very similar application to our proposed project because it also offers home cleaning service but the difference of our proposed project is that it only allows user to use one specific condo unit address because in goodwork.ph user can book a cleaning service anywhere wherein the safety of the people who cleans and users/customer are at risk. User can also use it for pranking someone or fake booking. Our proposed project is connected to different service provider agencies that offers cleaning service.
