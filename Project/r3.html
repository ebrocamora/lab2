Reflection 3

First Web Application using Github and Heroku

July 26, 2019

This week’s task is to create a web application using Github and Heroku. At first, I thought that it was too easy because I have a little background in HTML, but I was wrong. When I started creating my web application, I can’t achieve the design that I want for my web app. I still need to search in the internet on how to add padding in my content so that it will look like a very good web app. After that, I have a new problem and it is in the Github, I don’t know how to commit and push in the Github because it was my first time to use it. Now, I need to search again on how to create repository, commit, and push. I am so happy when my files appeared in the website of Github yeheeyyy hahahha. After committing and pushing in Github now I am in the phase of connecting my web app to Heroku.

The process was smooth until I reach the building part of my web application. It always says pushing problems, but I push it right because I saw in the youtube how to commit and push in Github. I got frustrated because I don’t know what to do but luckily one of my classmates (Christian Dinglasan) share to us that we need to change the file type of our HTML file into PHP (Thank you Ding!). When I did it, the building phase in Heroku was successful and my web application run in Heroku (yeeeheeeyyy). This week’s task makes me realize that my knowledge is not enough, and I need to do some effort for me to grow. It also makes me realize that self-exploration is good because you are learning by yourself and it feels good when you accomplish something that is worth it.
